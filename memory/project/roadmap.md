# 路线图

## 当前阶段

Structural Entropy Reduction（持续收口）

## 当前里程碑

首页逻辑态势图

## 里程碑成功标准

- 首页首屏不滚动即可回答当前目标、里程碑、阶段、风险和下一步
- 首页主视觉是可点击的逻辑结构图，而不是工程对象卡片堆叠
- `Kernel / Project` tab、artifact health、boundary groups 和常驻 runtime 状态板退出首页
- 不新增新状态源、重型框架或图形库

## 当前优先级

优先把首页收成人类友好的项目逻辑态势图，移除 `Kernel / Project` 工程视角和工程内部对象的首屏暴露。

## 下一阶段方向

- 在首页逻辑态势图稳定后，再继续压跨页面唯一 snapshot
- 继续压实 release 单一状态机，避免 task / release / runtime 重新长出平行口径
- 继续在不增加新状态源的前提下退休低价值解释层和重复外壳
